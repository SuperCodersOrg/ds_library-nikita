# Section 1 – Specific Bug

Today I encountered two different issues while working on the project.

The first issue happened during the initial Git repository setup. When I tried to push my local repository to GitHub, I received the following error:

```text
! [rejected] main -> main (fetch first)
error: failed to push some refs
```

This happened because the remote repository already contained commits, so Git prevented my local branch from overwriting the existing history.

The second issue was discovered after implementing the `DynamicArray<T>` using `malloc()` and `realloc()`. The implementation compiled successfully and worked correctly for primitive data types like `int`, but I realized that constructors and destructors were never being called. This meant the implementation was not safe for C++ objects such as `std::string` or user-defined classes.

---

# Section 2 – Failed Attempt

At first, I assumed that once the DynamicArray worked correctly with integers, the implementation would also work for any generic type.

I completed the implementation using `malloc()` and `realloc()`, tested all the operations, and initially thought everything was correct. After reading more about C++ memory management, I understood that these functions only allocate raw memory and do not construct or destroy objects. Although the code worked for primitive types, it was not a correct generic implementation.

Instead of continuing with this implementation, I decided to redesign the DynamicArray using **placement new**. The new implementation will allocate raw memory using `operator new`, construct objects using placement new, manually invoke destructors when objects are removed, and finally release the memory using `operator delete`. This approach will correctly support all C++ object types.

For the Git issue, my first attempt was simply running `git push`. Since the remote repository already contained commits, Git rejected the push. I then pulled the remote repository using `git pull --allow-unrelated-histories`, resolved the histories, and pushed the project successfully.

# Section 5 – Learning Reflection

Today's work taught me two important concepts.

The first was understanding the Git workflow. I learned that if a remote repository already contains commits, Git will reject a direct push to prevent overwriting history. Pulling the remote changes first and merging the histories solved the problem, and I now have a better understanding of how local and remote repositories stay synchronized.

The second lesson was about C++ memory management. Before today, I thought `malloc()` and `new` performed almost the same task because both allocate memory. After implementing the DynamicArray and reviewing how C++ objects are created, I realized that `malloc()` only reserves raw memory, while `new` also calls constructors. This helped me understand why generic C++ containers cannot rely on C-style memory allocation.

Because of this, I decided that my next implementation will use **placement new** along with `operator new` and `operator delete`. This will allow objects to be constructed and destroyed correctly while still giving me complete control over memory management. I also need to update my design proposal to reflect this change before continuing with the implementation.
